双指针（正确的）

左右同时指向一个数的时候，非0交换自己本身，左右同时向后移动，若为0，左不变停下来等待，右寻找非零元素，寻找到以后交换左右

  • 这里为什么会报错呢？

    在函数中使用了一个固定大小的数组 num[n]，并尝试返回它，但 C++ 中的数组不能以这种方式返回。可以用 std::vector 来替代

  1. 数组的返回限制：
    • C++ 中的数组在函数中是不能作为返回值直接返回的，因为数组名会被视为指针，而函数不能返回局部数组的地址（这会导致悬空指针问题）。
  2. 使用 std::vector 的优势：
    • std::vector 是动态数组，可以在运行时调整大小，且可以安全地作为返回值返回。
    • 它管理内存，可以自动处理内存分配和释放，避免手动管理内存带来的风险。
  3. 更简洁的语法：
    • 使用 std::vector 可以使代码更简洁和易于理解，因为它提供了许多有用的成员函数和操作。

因此，使用 std::vector 是一种更安全和灵活的选择，尤其是在需要动态管理数组大小时。这样可以避免许多潜在的错误和内存管理问题。

leetcode解答

将数组元素对应为索引的位置加n

遍历加n后的数组，若数组元素值小于等于n，则说明数组下标值不存在，即消失的数字

因为 1 <= nums[i] <= n，所以用 nums[i]-1 可以表示0~n的下标，对当前这个数字+n，此时这个数字会超过数组长度n，之后遍历凡是小于等于n的数组值，则证明原本的i+1的下标位置的数值不存在

[][][][]EG: [4] [2] [1] [1]

  • 4-1 = 3 原来4应该存放在[] [] [] [4]------下标为3
  • 1+4 = 5 数组变成[4] [2] [1] [5]
  • 重复上面操作：数组变成[12] [6] [1] [5]
  • 此时1<4 则这个下标为2 的数字应该是消失的数字2+1 = 3
