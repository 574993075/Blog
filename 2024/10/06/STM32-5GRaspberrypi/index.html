  • MX_ADC1_Init()：

    初始化 ADC1 外设：配置 ADC（模数转换器）1 的工作模式、分辨率、采样时间等参数。这个函数确保 ADC1 外设能够进行模数转换操作，以获取模拟信号的数字值。

  • MX_TIM1_Init()：

    初始化定时器 TIM1：配置 TIM1 定时器的计数模式、预分频器、计数周期、输出比较模式等。定时器用于时间测量、延迟生成、PWM 信号输出等功能。

  • MX_TIM3_Init()：

    初始化定时器 TIM3：与 TIM1 类似，配置 TIM3 定时器的计数模式、预分频器、计数周期等。这可能用于不同的定时任务或生成不同的 PWM 信号。

  • uint8_t a[1]={0x01};：

    定义数据数组：创建一个包含一个字节的数组 a，其值为 0x01。这通常用于准备要发送的数据。

  • HAL_UART_Transmit(&huart1, (uint8_t *)a, 1, 0xFFFF);：

    发送数据：使用 HAL 库函数通过 USART1 外设发送数据。这里，&huart1 是指向 UART_HandleTypeDef 结构体的指针，它包含 USART1 的配置和状态信息。(uint8_t *)a 是发送的数据源，1 是发送的数据长度（1 字节），0xFFFF 是超时时间，通常表示最长等待时间（可能是无限等待，直到数据完全发送完毕）。

  • MX_USART2_UART_Init()：

    初始化 USART2 外设：配置 USART2 的通信参数，如波特率、数据位、停止位、校验位等。这使得 USART2 外设能够进行串行通信，通常用于接收或发送数据。

    • 初始化 FreeRTOS 操作系统

      FreeRTOS是一个迷你的实时操作系统内核。作为一个轻量级的操作系统，功能包括：任务管理、时间管理、信号量、消息队列、内存管理、记录功能、软件定时器、协程等，可基本满足较小系统的需要

    • 是 FreeRTOS 中的一个函数，用于启动操作系统内核

      1. 启动调度器：osKernelStart() 函数启动 FreeRTOS 调度器，使系统进入多任务调度状态，从而开始任务的执行。调度器将开始按照任务的优先级和调度策略来管理任务的切换和执行。
      2. 执行任务：在调用 osKernelStart() 后，FreeRTOS 会按照你定义的任务优先级和调度算法来执行任务。所有在初始化时创建的任务将开始运行。
      3. 进入实时操作系统模式：函数调用后，FreeRTOS 会开始管理和调度任务，系统将从主程序的执行流切换到实时操作系统模式，执行预先配置的任务。

void SystemClock_Config

作用如下：原因

  • 配置系统时钟源，包括 HSE 和 PLL，使系统能够以特定的频率运行。

  • 设置 AHB 和 APB 总线的时钟分频，以确保外设的时钟频率在合理范围内。

  • 配置特定外设（如 ADC）的时钟源和分频。

void HAL_UART_RxCpltCallback

  • 处理串口通信中接收数据的中断

    1. 数据接收处理：
      • 当 USART1 或 USART2 这两个 UART 之一接收到数据时，中断会触发，调用这个回调函数来处理接收到的数据。
    2. 状态标志与缓冲区管理：
      • 代码根据 UART 的实例（USART1 或 USART2）选择适当的缓冲区和状态标志。
      • 使用状态标志来跟踪接收到的数据是否完成一个数据帧。
      • 处理接收到的字符，根据特殊字符（如回车 0x0D 和换行 0x0A）确定数据帧的结束

处理串口通信中的数据接收。它通过中断回调函数来处理数据，将接收到的数据存储到缓冲区，并根据特定的字符（回车和换行符）来判断数据帧的结束。这种方式适用于需要逐字节接收数据并在特定条件下完成处理的应用场景

void StartUART1Task

  1. 初始化 UART 接收：

    这两行代码启动了对 huart1 和 huart2 的 UART 接收中断。这意味着当接收到一个字节时，中断服务程序会被调用。

  2. 循环处理 UART 接收的数据：

    这是一个无限循环任务，任务会不断地检查是否接收到数据，并处理这些数据。

  3. 处理 UART1 接收的数据：

    当 UART1_RX_STA 标志位的高位设置时（表示接收完成），任务会处理接收到的数据，根据数据内容执行不同的操作。例如，命令包含 “read” 时，会启动传感器的周期性测量；命令包含 “stop” 时，会停止测量并切换 GPIO 状态。

  4. 处理 UART2 接收的数据：

    与处理 UART1 类似，任务会检查 UART2_RX_STA 标志位，并根据接收到的数据执行相应操作。

  • 主循环中通过 __HAL_TIM_SetCompare() 函数动态调整 PWM 的占空比，以改变电机的运行状态或速度。
  • 使用 osDelay(1) 实现任务的周期性挂起，以防止过高的 CPU 占用率

数据传输的格式/通信协议

1.起始位

当未有数据发送时，数据线处于逻辑“1”状态；先发出一个逻辑“0”信号，表示开始传输字符。

2.数据位

紧随起始位之后，数据位表示真正要发送或接收的信息，位数一般有8位或9位

3.奇偶校验位

数据位末尾可以选择是否添加奇偶校验位，用于检测数据传输是否正确

4.停止位

代表信息传输结束的标志位，可以是1位，1.5位或2位。停止位的位数越多，数据传输的速率也越慢。

初始化和配置 RCC（时钟控制器）寄存器来设置系统时钟源和分频因子

RCC_OscInitTypeDef 结构体初始化

  • OscillatorType: 设置要初始化的振荡器类型。在这里是 RCC_OSCILLATORTYPE_HSE，表示使用外部高速振荡器（HSE）。
  • HSEState: 设置 HSE 的状态。RCC_HSE_ON 表示开启 HSE。
  • HSEPredivValue: 设置 HSE 的预分频值。RCC_HSE_PREDIV_DIV1 表示不进行预分频。
  • HSIState: 设置 HSI（内部高速振荡器）的状态。RCC_HSI_ON 表示开启 HSI。
  • PLL.PLLState: 设置 PLL（相位锁环）的状态。RCC_PLL_ON 表示开启 PLL。
  • PLL.PLLSource: 设置 PLL 的时钟源。RCC_PLLSOURCE_HSE 表示 PLL 使用 HSE 作为输入源。
  • PLL.PLLMUL: 设置 PLL 的倍频因子。RCC_PLL_MUL9 表示 PLL 输出频率为 HSE 频率的 9 倍。

2. RCC_ClkInitTypeDef 结构体初始化

  • ClockType: 指定要配置的时钟类型。在这里，配置了 HCLK（AHB 总线时钟）、SYSCLK（系统时钟）、PCLK1（APB1 总线时钟）和 PCLK2（APB2 总线时钟）。
  • SYSCLKSource: 设置系统时钟源。RCC_SYSCLKSOURCE_PLLCLK 表示系统时钟源来自 PLL 输出。
  • AHBCLKDivider: 设置 AHB 总线时钟的分频因子。RCC_SYSCLK_DIV1 表示不分频。
  • APB1CLKDivider: 设置 APB1 总线时钟的分频因子。RCC_HCLK_DIV2 表示分频因子为 2。
  • APB2CLKDivider: 设置 APB2 总线时钟的分频因子。RCC_HCLK_DIV1 表示不分频。

3. RCC_PeriphCLKInitTypeDef 结构体初始化

  • PeriphClockSelection: 选择要配置的外设时钟。在这里是 RCC_PERIPHCLK_ADC，表示配置 ADC 外设的时钟。
  • AdcClockSelection: 设置 ADC 时钟的分频因子。RCC_ADCPCLK2_DIV6 表示 ADC 时钟分频因子为 6。

4. 函数调用与错误处理

  • HAL_RCC_OscConfig(&RCC_OscInitStruct): 配置振荡器。如果配置失败，调用 Error_Handler() 处理错误。
  • HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_2): 配置系统时钟和分频器。如果配置失败，调用 Error_Handler() 处理错误。
  • HAL_RCCEx_PeriphCLKConfig(&PeriphClkInit): 配置外设时钟。如果配置失败，调用 Error_Handler() 处理错误
